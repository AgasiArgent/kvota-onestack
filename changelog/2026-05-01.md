## Новое

- **Закупка завершается по КП, а не по всей заявке.** На каждом КП поставщика появилась своя кнопка «Завершить закупку по КП» — теперь не нужно ждать самого медленного поставщика, чтобы передать готовые позиции в логистику и таможню. Бейдж «Закупка завершена DD.MM» появляется на той карточке КП, которую закрыли. Шапочная кнопка «Завершить закупку» на всю заявку убрана.
- **Авто-движение карточек на канбане.** Карточка сама переезжает между колонками, когда менеджер делает соответствующее действие:
  - Распределение → Поиск поставщика — когда у бренда назначены МОЗ на все позиции
  - Поиск поставщика → Ожидание цен — когда КП помечен отправленным поставщику
  - Ожидание цен → Цены готовы — когда МОЗ нажал «Завершить закупку по КП» и все позиции бренда теперь покрыты завершёнными КП
  Каждый автопереход сопровождается уведомлением в углу: «Карточка автоматически переведена в …».
- **Канбан обновляется сам.** После любого действия (назначение, отправка, завершение) карточки переезжают и пересортируются без нажатия F5. Самые свежие — сверху.
- **«Отправлено поставщику»** — отдельная кнопка на КП-карточке. Помечает КП как отправленный без email-композера, заодно двигает канбан в «Ожидание цен».

## Улучшения

- **Очереди логистики и таможни отвязаны от уровня квоты.** Раньше всю работу можно было передать дальше только разом по всей заявке. Теперь каждый завершённый КП поставщика самостоятельно появляется в Unassigned-табе соответствующего отдела — head_of_logistics / head_of_customs назначает исполнителя вручную из своей очереди.
- **Сводки в очередях считают только релевантные КП.** Карточки «В работе у логистов / у таможни», «Просрочено», «Средний SLA» теперь учитывают только КП, по которым закупка уже завершена. Старые висящие назначения от предыдущей версии не зашумляют статистику.
- **Назначение исполнителей надёжнее.** Назначение/перевод логиста / таможенного теперь сразу проставляет timestamp начала работы (раньше иногда проставлялся только UUID — статистика «не видела» назначение).
- **Cargo places (грузовые места) можно создавать с пустыми значениями.** На уровне БД колонки веса/габаритов теперь nullable — позволяет добавить место и заполнить размеры позже, когда придут от поставщика.

## Исправления

- **Канбан закупок открывается с карточками.** Был случай, когда страница показывала пустые колонки из-за HTTP-301 редиректа на `app.kvotaflow.ru` — auth-токен терялся при переходе, и Python-API отвечал 401. Caddy перенастроен: `/api/*` на корневом домене теперь проксируется напрямую без редиректа.
- **Автозаполнение `quote_brand_substates` для новых заявок.** Карточки больше не «теряются» на канбане после создания квоты — пропавшие 17 квот восстановлены, на VPS прокачен ретро-апдейт стадий.
- **Уведомления на странице закупки в карточке заявки.** Toast'ы при действиях («Закупка завершена», «КП отправлен», ошибки) рендерились в никуда — на странице не был смонтирован Toaster. Починено.
- **Город в КП валидируется по справочнику.** Поле «Город отгрузки» теперь — автокомплит с подсказками из HERE-каталога (выпадает список при вводе, сохраняется каноничное название). Раньше можно было сохранить опечатку.
- **Модалки корректно прокручиваются на коротких экранах.** Шапка с заголовком и кнопки «Отмена / Создать» остаются видны, содержимое прокручивается между ними.
