## Новое

- **Заметки на сущностях.** Теперь можно вести заметки прямо у компании-покупателя и у конкретного КП поставщика — не только в карточке запроса целиком. Удобно фиксировать договорённости там, где они возникают.
- **Аналитика для руководителей.** Отдельная панель в воркспейсе для head_of_* ролей: сводные показатели по запросам, КП, закупкам и выручке.
- **Сохранённые виды таблиц для таможни.** Фильтры и сортировки можно сохранить как шаблон и переключаться между ними одним кликом.
- **CRUD шаблонов логистических маршрутов** в админке (`/admin/routing`) + inline-редактирование типа местоположения в реестре.
- **Smart-Delta флаги.** При правке позиций КП после согласования автоматически выставляется «требует проверки» — соответствующий менеджер видит изменения у себя в списке.
- **SLA-мониторинг в фоне** с защитой от дублирующих уведомлений.

## Улучшения

- **Создание КП поставщику.** Теперь обязательны только два поля: поставщик и компания-покупатель. Страну, город, инкотермс, валюту, грузовые места можно заполнить позже — после ответа поставщика. Это соответствует реальному рабочему сценарию.
- **Таблица позиций в КП переупорядочена.** Сначала идут поля из заявки продажника (бренд, артикул, наименование — только для чтения), потом два поля «артикул производителя» и «наименование производителя», которые заполняются закупкой только при замене товара поставщиком.
- **Модальные окна теперь корректно работают на коротких экранах.** Шапка с заголовком и кнопки «Отмена / Создать» остаются видны, содержимое прокручивается между ними. Применено ко всем модалкам в системе.
- **Расширенный модал позиции в таможне** с полным набором полей и расходами по товару.

## Исправления

- **КП поставщику: 5 ошибок** — некорректная нормализация страны/ставки НДС, потеря полей при определённых сценариях, сбои при повторном открытии карточки.
- **Убран дубль позиций над таблицей закупок.** Раньше для каждой позиции отображалась лишняя строка-дубликат; теперь там осталась только индикация split/merge для позиций, где она применима.
- **Реестр локаций**: устранена ошибка из-за отсутствующей колонки `country_iso2`.
