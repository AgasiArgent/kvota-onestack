## Caveats for implementation

1. **Roles in `data.js` include `sales_manager` and `currency_controller`** ‚Äî these were legacy roles removed in migration 168. When translating to production, use only the 13 roles currently defined in `.kiro/steering/access-control.md`. Replace:
   - `sales_manager` ‚Üí `sales` (or `head_of_sales` depending on context)
   - `currency_controller` ‚Üí `finance`

2. **Pin coordinates are relative (0.0‚Äì1.0)** ‚Äî this matches the spec (Requirement 8.3 stores `last_rel_x`, `last_rel_y`, `last_rel_width`, `last_rel_height`). Do NOT convert to pixels.

3. **Cluster colour palette** in `data.js` is a starting point ‚Äî the production implementation should pull from `design-system.md` tokens, not hardcode these hex values.

4. **`design-canvas.jsx`** is a presentation-only wrapper (pan/zoom + post-it notes). Production `/journey` does NOT need this ‚Äî it uses React Flow directly via our own components. This file is useful only if we want a "design review mode" later.

5. **The 16 nodes in `data.js`** are a representative slice, not the full 34. Production manifest generator will produce all 34 (Requirement 17.1).

6. **Shell menu** (`shell.jsx`) shows `/journey` with `active: true, isNew: true`. When wiring into the real `widgets/sidebar`, add it to "–ì–ª–∞–≤–Ω–æ–µ" section with a "NEW" badge for the first release cycle.

## Relationship to the spec

This mockup is **visual reference** ‚Äî the authoritative contract is:
- `.kiro/specs/customer-journey-map/requirements.md` (18 requirements, ~80 EARS AC)
- `.kiro/specs/customer-journey-map/design.md` (architecture + type contracts)
- `docs/superpowers/specs/2026-04-22-customer-journey-map-design.md` (rationale)

When mockup and spec disagree, the spec wins. Deviations worth noting as you implement:
- `data.js` roles ‚Äî see caveat 1 above
- Mockup has no explicit concurrency UI ‚Äî spec Requirement 6.1‚Äì6.3 requires `version` field with 409 STALE_VERSION handling
- Mockup has no explicit orphan-annotations panel ‚Äî spec Requirement 13 requires it (admin-only sidebar section)
