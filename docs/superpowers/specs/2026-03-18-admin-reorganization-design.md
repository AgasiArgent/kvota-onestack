# Admin Pages Reorganization — Design Spec

**Date:** 2026-03-18
**Status:** Approved

## Overview

Реорганизация admin-страниц OneStack для логичной группировки: справочники выносятся в «Реестры», маршрутизация закупок объединяется в одну страницу с 4 вкладками, доступ расширяется за пределы admin-only.

## Changes Summary

### 1. Sidebar — перенос справочников в «Реестры»

**Юрлица** (`/companies`) — из Администрирования в Реестры.
- Доступ: admin, finance, procurement (ранее admin-only)

**Локации** (`/locations`) — из скрытого в Реестры.
- Доступ: admin, logistics, customs, procurement

### 2. Маршруты закупок — объединение в одну страницу

Текущие `/admin/brands` и `/admin/procurement-groups` объединяются в единую страницу `/admin/routing` с 4 вкладками.

**Название в sidebar:** «Маршруты закупок»
**Доступ:** admin, head_of_procurement (ранее admin-only)

#### Вкладка 1: По брендам
- Бренд → конкретный закупщик
- Автоматическое назначение при поступлении заявки
- При смене закупщика в старых сделках остаётся прежний МОЗ (история)
- Импорт неназначенных брендов из существующих КП
- **Перенос функционала из `/admin/brands`**

#### Вкладка 2: По группам продаж
- Группа продажников → закупщик
- МОП из группы А создал КП → позиции идут на закупщика группы А
- **Перенос функционала из `/admin/procurement-groups`**

#### Вкладка 3: Тендерные
- Специальная цепочка для тендерных сделок
- Заявка → тендерный отдел → руководитель → распределённый МОЗ
- Настраиваемая цепочка ответственных
- **Новый функционал**

#### Вкладка 4: Нераспределённые (диспетчер)
- Очередь заявок, не попавших ни под одно правило
- Руководитель закупок назначает вручную на конкретного МОЗ
- Мультибрендовые заявки попадают сюда автоматически
- При назначении — чекбокс «Закрепить бренд за этим МОЗ» (создаёт правило на будущее)
- **Новый функционал**

### 3. Приоритет правил маршрутизации

При поступлении заявки проверяется каскадно (первое совпадение выигрывает):

1. **Тендер?** → тендерная цепочка
2. **Группа продаж назначена?** → МОЗ по группе
3. **Мультибренд?** → нераспределённые (ручное)
4. **Бренд назначен?** → МОЗ по бренду
5. **Ничего не подошло** → нераспределённые (ручное)

### 4. Удаление из sidebar

- **Журнал звонков** (`/calls`) — drop (Mango не интегрирован)

### 5. Без изменений

- **Пользователи** (`/admin`) — остаётся как есть
- **Обращения** (`/admin/feedback`) — остаётся как есть
- **Настройки** (`/settings`) — остаётся как есть (только calc rates)
- **Impersonation** (`/admin/impersonate`) — остаётся скрытым (dev tool)

## Implementation Scope

**Phase 1 (sidebar + простые переносы):**
- Перенести Юрлица и Локации в Реестры (sidebar config change)
- Убрать Журнал звонков из sidebar
- Расширить доступ к Юрлицам и Локациям по ролям

**Phase 2 (маршруты закупок):**
- Объединить brands + procurement-groups в одну страницу с вкладками
- Мигрировать на Next.js
- Расширить доступ для head_of_procurement

**Phase 3 (новый функционал):**
- Вкладка «Тендерные» — настройка цепочки
- Вкладка «Нераспределённые» — очередь диспетчера + закрепление брендов
- Интеграция каскада приоритетов в workflow назначения
