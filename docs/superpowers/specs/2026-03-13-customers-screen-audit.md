## Что пользователь реально делает здесь

### Список клиентов (`/customers`)
1. **Найти клиента** по названию или ИНН (основное действие — 80% визитов)
2. Посмотреть сводку: сколько клиентов, сколько активных, сколько с контактами
3. Создать нового клиента
4. Перейти в карточку клиента

### Карточка клиента (`/customers/{id}`)
1. **Посмотреть реквизиты** (ИНН/КПП/ОГРН) — для выставления документов
2. **Посмотреть контакты** — кому звонить/писать
3. **Посмотреть историю КП** — что уже предлагали этому клиенту
4. **Создать новый КП** для этого клиента
5. Назначить/сменить менеджера

## Текущая структура табов (10 штук!)

| Таб | Содержание | Реально нужен? |
|-----|-----------|----------------|
| **Общая** | Реквизиты + превью контактов/договоров + задолженность + таблицы КП/Спеков | Да, но перегружен |
| **Адреса** | Юр/Факт/Почтовый адрес + склады | Редко используется |

| **Позиции** | Запрошенные позиции по всем КП | Узкоспециальный |
| **Ещё** | Одно поле "Заметки/Примечания" | Целый таб для одного поля?! |
| **Звонки** | Журнал звонков с этим клиентом | Нужен |

## Проблемы

### Список
1. **Длинные названия ломают таблицу** — названия типа "АНОО ВО УНИВЕРСИТЕТ СИРИУС..." занимают 3 строки
2. **Нет пагинации** — 156 клиентов рендерятся все сразу (пока ок, но будет тормозить)
3. **Колонка "Менеджер" часто пустая** — у многих клиентов "—"
4. **Нет колонки "Кол-во КП"** — не видно кто активный клиент, а кто пустая запись

### Карточка
1. **10 табов — слишком много.** Когнитивная перегрузка. Пользователь не знает где искать
2. **Таб "Ещё" — для одного поля "Заметки".** Абсурдно, заметки должны быть на основной вкладке
3. **Дублирование КП/Спеки:** и на вкладке "Общая" (превью), и отдельные табы
4. **Таб "Общая" перегружен:** реквизиты + контакты-превью + договоры-превью + задолженность + КП + Спецификации — всё на одном экране
5. **Inline-редактирование "кликните для редактирования"** — не очевидно для пользователя что поля кликабельны
6. **Менеджер как дропдаун с 30+ именами** — трудно найти нужного, нет поиска
7. **Адреса — отдельный таб** для 3 строчек текста, которые редко меняются

## Предложение по упрощению

### Список клиентов
- Добавить колонку "КП" (количество) — сразу видно активных клиентов
- Обрезать длинные названия с `...` (max ~50 символов)
- Добавить колонку "Последний КП" (дата) — показывает активность

### Карточка: сократить 10 табов → 4

| Новый таб | Что включает | Откуда |
|-----------|-------------|--------|
| **Обзор** | Реквизиты + менеджер + задолженность (компактно) + счётчики КП/Спеков | Общая (упрощённая) |
| **CRM** | Адреса + Контакты + Звонки + Встречи + Заметки (оптимизированно) | Адреса + Контакты + Звонки + Встречи + Ещё |
| **Документы** | КП + Спецификации + Договоры | КП + Спеки + Договоры |
| **Позиции** | Запрошенные позиции по всем КП | Позиции (без изменений) |

**Что упрощается:**
- 10 табов → 4 таба
- "Ещё" (заметки) → переехало в CRM
- Адреса → переехали в CRM (вместе с контактами — всё про коммуникацию)
- КП + Спеки + Договоры → один таб "Документы"
- Звонки + Встречи → в CRM (вместе с контактами)
- "Общая" → упрощённый "Обзор" без дублирования таблиц

### Таб "Обзор" — layout

```
┌─────────────────────────────────────────────────────────┐
│ АО "АЛВИЗ"                          Активен  [+ КП]    │
│ Менеджер: Савостина С.               ИНН: 2900000293   │
├────────────────────────┬────────────────────────────────┤
│ Реквизиты              │ Задолженность                  │
│ ИНН   2900000293       │ Долг: 0 ₽                      │
│ КПП   290101001        │ Неоплачено: 0 позиций          │
│ ОГРН  1022900516110    │ Нет просрочек                  │
│ Источник: —            │                                │
├────────────────────────┴────────────────────────────────┤
│ КП                              │ Спецификации          │
│ На рассмотрении: 2              │ Активных: 1           │
│ В подготовке: 1                 │ Подписанных: 3        │
│ Всего: 12        [Все →]       │ Всего: 4   [Все →]   │
└─────────────────────────────────────────────────────────┘
```

- КП и Спеки показаны как **компактные счётчики**, не таблицы
- Нет дублирования — полные таблицы живут в табе "Документы"
- Задолженность — компактный блок справа

### Таб "CRM" — layout

```
┌─────────────────────────────────────────────────────────┐
│ [Контакты]  [Адреса]  [Звонки]  [Встречи]  [Заметки]  │
├─────────────────────────────────────────────────────────┤
│ Секция по выбранному подтабу                            │
│ (Контакты — таблица, Адреса — карточки,                │
│  Звонки/Встречи — хронология, Заметки — textarea)      │
└─────────────────────────────────────────────────────────┘
```

- Внутри CRM: подтабы-переключатели (pills/segments), не отдельные страницы
- Всё про коммуникацию с клиентом в одном месте

### Таб "Документы" — layout

```
┌─────────────────────────────────────────────────────────┐
│ [КП]  [Спецификации]  [Договоры]                        │
├─────────────────────────────────────────────────────────┤
│ Таблица выбранного типа документа                       │
│ (с фильтром по статусу, сортировкой по дате)           │
└─────────────────────────────────────────────────────────┘
```
