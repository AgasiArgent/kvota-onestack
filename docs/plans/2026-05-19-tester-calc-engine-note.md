# Расчётный движок — записка тестеру (2026-05-19)

Прогнали движок Kvotaflow против эталонных Excel-форм. **Сходится в пределах ≤0.01%.** Покрытие — golden-suite в CI, гейтит каждый PR (сломал — не смержить).

## Что починили

- **Зона НДС** для поставщиков с метками «ЕС (между странами ЕС)» / «ЕС (закупка между странами ЕС)» определялась неправильно (всплыло на idemitsu). Исправлено.
- **Формула % DM-fee** на форме `forma` считалась не так. Исправлено.
- **Округление денежных ячеек** в движке выровняли под эталон — 2 знака после запятой, как и было в форме.

## Что НЕ баг (особенности эталона, не движка — репортить не нужно)

1. **Логистика по сегментам.** Kvotaflow бьёт логистику на переменное число сегментов; в эталоне всегда 3. Цифры по конкретным сегментам / по позиции внутри сегмента **могут отличаться** — это норма. Критерий — **итого по логистике совпадает**.
2. **idemitsu, остаточные ~0.039% на ~4 ячейках.** Внутренняя несогласованность округления в самом эталонном файле (ячейка Y16 пересчитывает по неокруглённому значению, а AX16 — округлённое). Движок считает правильно.
3. **forma, ~1.086% по финансовому блоку (BH2).** В этом эталонном файле **не указан продавец**, поэтому формула эталона не включает 22% НДС. Движок учитывает НДС — это правильно.

## Как проверить самому

Для любой квоты можно собрать **сравнительный Excel** — это файл-копия формы расчёта числами движка + два листа сравнения с эталоном по ячейкам. Скажи название квоты — сгенерируем и отдадим.

## Что репортить

Если расхождение с эталоном встретится **где-то ещё** (не в трёх случаях выше) — это баг, репорти в Testing 2 как обычно.
