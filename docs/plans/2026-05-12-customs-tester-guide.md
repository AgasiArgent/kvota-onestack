## Известные ограничения (НЕ баги)

### 1. Multi-supplier multi-country → колонка «Страна» пустая
Если один и тот же товар покрыт инвойсами от разных поставщиков из разных стран (Q-202605-0004: 4 товара под 3 КПП ADLER/CHONGQING), колонка «Страна» в плоской таблице пуста.

**Refactor запланирован** — `.kiro/specs/customs-per-invoice-refactor/`. Таблица станет per-КПП с чипами сверху, как procurement-step. Шелф пока, в будущем.

### 2. Автоматизация через Alta включится позже
Все Alta-driven кнопки и баннеры скрыты build-time флагом `NEXT_PUBLIC_ALTA_FEATURES_ENABLED`. Когда продукт решит включать — env-флаг + redeploy. Это отдельная история.

### 3. «Этап не завершается» (Олежкин баг)
Параллельный баг в его FB-фидбэке: после нажатия «Завершить закупку» с заполненными ТН ВЭД, этап не двигается. Это workflow-transition bug, **не связан с save-багом**. Будет диагностирован отдельно.

## Что важно ре-тестить после PR #141 (для Олега / Дениса)

### FB-260511-212235-0384 — save flow
1. Открыть КП на customs step
2. Открыть expand первой строки → dialog
3. Ввести ТН ВЭД (например `4002 31 000 0`)
4. Нажать «Сохранить»
5. **Ожидание:** dialog закрывается + toast «Данные по позиции сохранены» + значение в table cell обновлено
6. Refresh страницы → значение persists
7. **Ранее:** dialog не закрывался, table не обновлялась → silent failure

### Handsontable inline edit
1. Кликнуть на ячейку ТН ВЭД в таблице напрямую (без открытия dialog)
2. Ввести значение → Enter
3. **Ожидание:** значение сохраняется, без ошибки
4. Refresh → persists

### License cost migration
1. Открыть dialog → блок «Лицензии и разрешения»
2. **Должно быть:** только чекбоксы «требуется» для ДС/СС/СГР (без полей стоимости)
3. Закрыть dialog → блок «Сертификаты / Расходы» внизу страницы
4. Создать ад-хок expense → указать тип «Стоимость ДС/СС/СГР» + сумма + прикрепить к позиции
5. Стоимость должна разойтись между прикреплёнными позициями
