# Сегодняшний цикл — записка тестеру (2026-05-21)

Перед проверкой — `Ctrl+Shift+R` (Hard Refresh), иначе бандл старый.

Большинство изменений сегодня — под капотом (схема БД + единый формат ошибок API). Снаружи почти не видно. Но **два пре-существующих бага с тостами** мы заодно поймали и пофиксили — нужно перепроверить.

## ✅ Видимые фиксы

| Где | Что было | Что стало |
|---|---|---|
| Канбан закупок → попап «Назначить исполнителя» | При ошибке тост показывал `[object Object]` | Тост показывает реальное сообщение от сервера |
| Распределение закупок → карточка бренда → «Назначить» | То же `[object Object]` | То же — теперь читаемый текст |

## 🔍 Нужно от тебя

Прогони сценарии, где **ожидается ошибка**, и убедись, что **нигде не вылезает `[object Object]`** или сырой JSON. Где смотреть:

- **Расчёт КП** без позиций → должно быть «Cannot calculate - no products in quote» (или похожее), не `[object Object]`
- **Отмена КП** без указания причины / без прав → понятный текст
- **Передача в закупки** с незаполненным чеклистом → подсказка «Заполните контрольный список…»
- **Любая ошибка от API** (например, дёрнуть действие, на которое нет прав) — тост должен быть осмысленным

Если хоть раз увидишь `[object Object]`, `undefined` или просто пустую плашку при ошибке — **скриншот + где жал**. Это баг под отдельную задачу.

## 📋 Под капотом — не баг

- В БД дропнули две дублирующие колонки на `quotes` / `quote_items`. На UI это не видно: данные брались с правильной колонки и до этого.
- Все ошибки API теперь приходят в одной форме `{success: false, error: {code, message}}`. На UI это тоже не видно — фронт парсит и старый, и новый формат.
