## ✅ Row 31 — выпадающий Ед. в Заявка/таблице позиций

Починили окончательно. Открываешь Ед. → видишь все 13 вариантов: `шт / упак / компл / кг / г / т / м / мм / см / м² / м³ / л / мл`. На проде проверил скриншотом.

**Что было:** Handsontable v17 внутри табы оборачивает таблицу в `.ht-wrapper` с `overflow: clip` высотой 59 px (под заголовки колонок). Выпадашка позиционируется absolute, но всё равно обрезалась этим контейнером — DOM показывал 393 px высоты, а на экране ты видел 0–30 px.

**Что починили:** Сняли клип безусловно с этого враппера (главная таблица скроллится через внутренний контейнер, ни на что не повлияло).

Пришлось три захода (PRs #210/#211/#212), потому что:
1. Сначала пробросили `themeName` — починили только ширину
2. Forsили высоту — но клип-родитель резал
3. Поставили `:has()` селектор — но Tailwind 4 / lightningcss бандлер выбрасывал `:has()` из прод-бандла
4. PR #213 — безусловный override, заработало
