# PHMB Standalone Flow — Plan

**Date:** 2026-03-15
**Scope Mode:** HOLD SCOPE
**Status:** Ready for implementation

## Problem Statement
PHMB (price-list mode) — самостоятельный бизнес-процесс расчёта КП по прайс-листам. Сейчас встроен в общий quote detail на FastHTML. Нужно выделить в отдельный Next.js flow: от создания КП до отправки PDF клиенту. Это первый полный бизнес-цикл на Next.js — если сработает, станет шаблоном для миграции master flow.

## Scope Mode Rationale
Flow уже определён и работает на FastHTML. Задача — перенести с UX улучшениями, не придумывать новое. Две добавки: версионность КП и auto-calc вместо ручной кнопки.

## Роли
- **Менеджер по продажам** — создаёт КП, добавляет позиции, настраивает условия, формирует PDF
- **Закупщик** — устанавливает цену, срок, валюту на позиции без цены в базе
- **Админ** — настраивает overhead costs и скидки (уже DONE на /settings)

## Экраны (3 страницы)

### 1. `/phmb` — Реестр КП
- Список PHMB-КП менеджера (таблица, shadcn)
- Кнопка "Создать КП" → открывает диалог
- Диалог: клиент, валюта, дата → создать → redirect на `/phmb/[id]`

### 2. `/phmb/[id]` — Рабочее пространство КП
Главный экран. Одна страница, несколько секций:

**Header:**
- Номер КП, клиент, статус, дата

**Поиск и добавление позиций:**
- Поиск по артикулу/названию в `phmb_price_list`
- Если цена есть → сразу подставляется, позиция рассчитывается автоматически
- Если цены нет → позиция добавляется как "ожидает цену", уходит в очередь закупщика
- Таблица позиций: **Handsontable** (Excel-like: add/delete rows, copy-paste, cell navigation)
- Позиции с ценой показывают рассчитанную цену сразу
- Позиции без цены — отмечены визуально (ожидает закупщика)

**Условия КП (панель):**
- Аванс % (дефолт из settings)
- Отсрочка дней (дефолт из settings)
- Наценка % (дефолт из settings)
- При изменении условий → все позиции пересчитываются в реальном времени

**Auto-calc логика:**
- Нет кнопки "Рассчитать"
- Цена позиции считается автоматически: закупочная цена + overhead costs (из settings) + условия КП
- Формула из `phmb_calculator.py`, вызывается через Python API
- Пересчёт при: добавлении позиции, изменении условий, получении цены от закупщика

**Версионность:**
- Кнопка "Создать версию" — копирует позиции с текущими условиями
- Менеджер может создать несколько версий с разными условиями (100% аванс vs отсрочка 30 дней)
- Переключение между версиями
- Каждая версия → свой PDF

**Partial quotes:**
- Менеджер видит: 7 из 10 позиций с ценой
- Может сформировать КП из позиций с ценой прямо сейчас
- Или подождать пока закупщик оценит остальные

**PDF экспорт:**
- Кнопка "Скачать PDF" — генерация коммерческого предложения
- Python API endpoint (серверная генерация)
- Экспортирует текущую версию с текущими условиями

### 3. `/phmb/procurement` — Очередь закупщика
- Список позиций без цены, сгруппированные по brand group
- Закупщик устанавливает: цену, срок поставки, валюту
- После установки цены → позиция автоматически рассчитывается в КП менеджера
- Telegram уведомление менеджеру (существующий сервис, добавить route)

## Таблицы БД (существующие, без миграций)
- `phmb_price_list` — прайс-лист (read-only)
- `phmb_quote_items` — позиции КП
- `phmb_settings` — overhead costs организации
- `phmb_brand_type_discounts` — скидки по брендам
- `phmb_brand_groups` — группы брендов
- `phmb_procurement_queue` — очередь закупщика
- `quotes` — КП (subtotal, total_amount)
- `quote_versions` — версии (если есть в БД)

## Технический стек
- **Frontend:** Next.js 15 + shadcn/ui + Tailwind + Handsontable (для таблицы позиций)
- **Data:** Supabase direct (CRUD) + Python API (расчёт + PDF)
- **Calc:** Python API endpoint → `phmb_calculator.py`
- **PDF:** Python API endpoint (серверная генерация)
- **Notifications:** Существующий Telegram сервис + новый route
- **Table component:** Handsontable React (Excel-like UX для позиций)

## Core Requirements
1. Менеджер может создать PHMB КП (клиент + валюта)
2. Менеджер может искать позиции по артикулу/названию в прайс-листе
3. Позиции с ценой в базе — цена подставляется автоматически, расчёт мгновенный
4. Позиции без цены — уходят в очередь закупщика, отмечены как "ожидает цену"
5. Закупщик видит очередь, устанавливает цену + срок + валюту
6. После установки цены закупщиком — позиция автоматически рассчитывается в КП
7. Менеджер может настроить условия КП (аванс %, отсрочка, наценка %) — пересчёт в реальном времени
8. Менеджер может создать версию КП с другими условиями
9. Менеджер может переключаться между версиями
10. Менеджер может сформировать partial quote (только позиции с ценой)
11. Менеджер может скачать PDF коммерческого предложения
12. Telegram уведомление менеджеру когда закупщик установил цену
13. Таблица позиций — Handsontable (add/delete rows, copy-paste, cell navigation)
14. Admin-only настройки PHMB — DONE (`/settings` tab)
15. PHMB toggle на профиле пользователя

## Deferred Items
1. **История изменений цен** — audit trail, не MVP
2. **Сравнение версий side-by-side** — переключение достаточно для MVP
3. **Bulk import позиций из Excel** — ручной ввод + copy-paste через Handsontable покрывает
4. **Мобильная версия таблицы** — Handsontable плохо работает на mobile, defer
5. **AG Grid для финансовых таблиц** — отдельная задача, не PHMB scope

## Key Decisions
- **3 экрана, не 6:** Создание КП = диалог (не страница). Поиск + позиции + условия + расчёт + результат = одна рабочая страница. Очередь закупщика = отдельная. Основано на proximity principle из refactoring-ui.
- **Auto-calc, не кнопка:** Цены считаются автоматически при добавлении позиции или изменении условий. Дефолты из settings, менеджер переопределяет per-quote.
- **Handsontable для позиций:** Excel-like UX (пользователи просили). AG Grid — для будущих финансовых таблиц (data grid сценарий).
- **Версионность через существующую модель в БД:** Версия = набор условий + рассчитанные цены. Позиции общие, условия per-version.
- **Отдельный flow, не таб:** `/phmb/*` — свои URL, свои экраны. Не зависит от основного quote detail. Proof of concept для миграции master flow.
